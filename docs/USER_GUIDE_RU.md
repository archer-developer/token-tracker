# Руководство пользователя - Трекер коммерческих токенов Беларуси

Приложение для отслеживания инвестиций в токенизированные облигации на платформах Fainex и Finstore.

## Структура приложения

Приложение имеет 5 основных разделов:

1. **Портфель** (Portfolio) — агрегированные метрики всех инструментов
2. **Инструменты** (Instruments) — CRUD операции с облигациями
3. **Календарь** (Calendar) — выплаты по месяцам
4. **Журнал** (Ledger) — полная история всех денежных потоков
5. **Настройки** (Settings) — конфигурация приложения

Все данные хранятся локально в браузере (IndexedDB). Резервная копия доступна в Настройках.

## Дашборд Портфеля

### Описание

Главная страница приложения показывает сводку по всему портфелю инвестиций.

### Метрики на дашборде

#### 1. Общая инвестированная сумма (Total Invested)

**Формула:** Сумма всех закупленных токенов по всем инструментам

```
Total Invested = Σ (quantity × price_per_token) для всех покупок
```

**Пример:** Если вы купили:

- 100 токенов GURMINA по 100 USD = 10,000 USD
- 50 токенов COMPASS по 150 USD = 7,500 USD

**Total Invested = 17,500 USD**

#### 2. Получено доход (Total Paid)

**Формула:** Сумма всех полученных выплат (купонов и возвратов)

```
Total Paid = Σ платежи со статусом "Выплачено"
```

Включает:

- Купонные платежи (coupon payments)
- Возвраты после дефолта (recovery payments)

**Пример:** Если получено 500 USD купонов + 200 USD возврата = 700 USD

**Total Paid = 700 USD**

#### 3. Текущие убытки (Current Loss)

**Формула:** Потери от дефолтов инструментов

```
Current Loss = Σ principal от дефолтивших инструментов (если recovery < 100%)
```

**Пример:**

- Инструмент A дефолтировал с основной суммой 5,000 USD
- Ожидается возврат только 2,000 USD (40%)
- Убыток = 5,000 - 2,000 = 3,000 USD

**Current Loss = -3,000 USD**

#### 4. Прибыль/убыток (P&L)

**Формула:**

```
P&L = Total Paid + Current Loss - Total Invested
или
P&L = (Полученные платежи + Ожидаемый возврат) - Инвестировано
```

**Пример:**

- Total Invested: 17,500 USD
- Total Paid: 700 USD
- Current Loss: -3,000 USD

**P&L = 700 + (-3,000) - 17,500 = -19,800 USD** (убыток)

#### 5. XIRR (Internal Rate of Return)

**Что это:** Внутренняя норма доходности всего портфеля в годовых процентах.

**Как считается:** Используется метод Ньютона-Рафсона для решения уравнения:

```
0 = Σ (cash_flow_i / (1 + XIRR)^(days_i / 365))
```

Где:

- `cash_flow_i` — денежный поток (отрицательный для инвестиций, положительный для доходов)
- `days_i` — дни от первого потока до i-го потока

**Пример:**

- 01.01.2025: Инвестировано -10,000 USD
- 15.03.2025: Получено купон +500 USD
- 01.04.2025: Получено купон +500 USD
- ...дальше прогноз на будущие платежи

**XIRR = 5.2%** (годовая доходность)

#### 6. Осталось получить (Total Remaining)

Две части:

**a) Купоны (Remaining Coupons)**

```
Remaining Coupons = Σ (expectedAmount всех запланированных купонов)
```

**b) Основная сумма (Remaining Principal)**

```
Remaining Principal = Σ (невыплаченная основная сумма при погашении)
```

**Пример:**

- Осталось получить купонов: 3,000 USD
- Осталось получить основной суммы: 8,500 USD
- **Total Remaining = 3,000 + 8,500 = 11,500 USD**

#### 7. График портфеля (Portfolio Chart)

Интерактивный график показывает:

- **Синяя линия** — историческая прибыль/убыток (до сегодня)
- **Оранжевая линия** — прогноз будущей прибыли/убытка

**Как считается:**

- **Исторический P&L**: Полученные платежи + Убытки от дефолтов − Начальные инвестиции
- **Прогноз**: Добавляются все будущие запланированные платежи

## Управление инструментами

### Добавление инструмента

1. Нажимаем **"Инструменты"** в меню
2. Кнопка **"Добавить инструмент"**
3. Заполняем форму:
   - **Название** — наименование облигации (например, GURMINA.USD.2024.01)
   - **Платформа** — Fainex или Finstore
   - **Валюта** — USD или EUR
   - **Ставка купона** — годовые проценты
   - **Стоимость токена** — цена за один токен при покупке
   - **Дата начала** — первый день обращения облигации
   - **Дата погашения** — день выплаты основной суммы
   - **Периодичность выплат** — ежемесячно, ежеквартально и т.д.

### Извлечение данных из White Paper

Если у вас есть PDF файл с условиями облигации:

1. Нажимаем **"Извлечь из White Paper (PDF)"**
2. Загружаем файл
3. Приложение автоматически заполнит поля на основе анализа текста

_Требует настройки LLM в разделе Настройки_

### Статусы инструмента

- **Активный** (Active) — облигация торгуется, выплаты идут по графику
- **Созревший** (Matured) — дата погашения прошла, основная сумма выплачена
- **Дефолт** (Defaulted) — эмитент не выплатил в срок

## Ввод платежей

### Лоты покупки (Purchase Lots)

Каждую покупку нужно записать как отдельный лот:

**Пример:** Инструмент куплен дважды

- Лот 1: 09.01.2025 — 100 токенов по 100 USD = 10,000 USD
- Лот 2: 26.01.2025 — 50 токенов по 102 USD = 5,100 USD

**Зачем нужны лоты:**

- Правильный расчет выплат (если покупка была после начала периода)
- Отслеживание XIRR по времени
- Корректное заполнение журнала

### Статусы выплат

На экране "Выплаты" каждая ожидаемая выплата имеет статус:

- **Запланировано** (Scheduled) — выплата еще не произошла
- **Выплачено** (Paid) — деньги получены
- **Пропущено** (Missed) — выплата просрочена

### Отметить как выплаченную

1. На вкладке "Запланировано" ищем выплату
2. Нажимаем **"Отметить как выплаченную"**
3. Заполняем форму:
   - **Фактическая сумма** — сколько реально получилось (может отличаться от ожидаемой)
   - **Дата выплаты** — когда поступили деньги

### Отметить как пропущенную

1. Нажимаем кнопку с иконкой ✗
2. **Подтверждаем** действие в диалоге
3. Выплата переместится в раздел "История"

## Календарь выплат

Интерактивный календарь по месяцам, показывает:

- **Все ожидаемые выплаты** по каждому месяцу
- **Сумму выплат** по месяцам
- **Статус каждой выплаты** (запланирована, выплачена, пропущена)

**Вкладки:**

- **По инструментам** — выплаты сгруппированы по облигациям
- **По типам** — купоны отдельно, погашение отдельно

## Журнал денежных потоков

Полная история всех операций в хронологическом порядке:

**Типы операций:**

- **Покупка** (Purchase) — отрицательное значение, отток денег
- **Купон** (Coupon) — положительное значение, приход дохода
- **Погашение** (Redemption) — положительное значение, возврат основной суммы
- **Возврат после дефолта** (Recovery) — положительное значение, частичный возврат после дефолта
- **Продажа** (Sale) — положительное значение, выручка от продажи токенов

## Настройки

### Тема (Theme)

- **Светлая** — для дневного использования
- **Темная** — для ночного использования
- **Система** — следит за настройками ОС

### Язык (Language)

- **Русский** — по умолчанию
- **Беларусский** — альтернативный язык интерфейса

### Валюта для отображения (Base Currency)

Все суммы будут конвертироваться в выбранную валюту:

- **BYN** — белорусские рубли (через курс NBP)
- **USD** — доллары (курс обновляется автоматически)
- **EUR** — евро (курс обновляется автоматически)

### Обновление курсов (Exchange Rates)

- **Автоматическое** — приложение проверяет курсы дважды в день (до полудня и после)
- **Ручное** — можно обновить курсы нажатием кнопки

Курсы берутся с официального API Национального банка Республики Беларусь (NBRB).

### LLM интеграция

Для автоматического извлечения данных из White Paper:

1. Введите **Base URL** — адрес LLM сервера (например, https://api.openai.com/v1)
2. Введите **API Key** — ключ доступа
3. Выберите **Model** — модель LLM (например, gpt-4o-mini)
4. Нажмите **"Тест"** для проверки соединения

### Резервная копия (Backup)

- **Экспортировать** — загружает JSON файл со всеми данными
- **Импортировать** — восстанавливает данные из JSON файла

_Используйте для переноса данных между устройствами_

### Режим презентации (Presentation Mode)

Загружает демо-данные для презентаций и тестирования. Не используется для реальных данных.

## Часто задаваемые вопросы

### Где хранятся мои данные?

Все данные хранятся локально в базе данных браузера (IndexedDB). Сервер не имеет доступа к вашим данным.

### Как экспортировать данные?

Перейти в Настройки → Резервная копия → **Экспортировать**. Будет загружен JSON файл со всеми данными.

### Как изменить валюту всех сумм?

Перейти в Настройки → **Валюта для отображения** → выбрать нужную. Все суммы пересчитаются автоматически.

### Почему XIRR показывает отрицательное значение?

XIRR отрицательный, когда убытки от дефолтов или низкие купоны больше, чем доход от инвестиций.

### Что означает "Осталось выплатить"?

Это сумма всех денег, которые вы должны получить в будущем:

- Купоны до даты погашения
- Основная сумма при погашении
- Возвраты после дефолта (если инструмент дефолтировал)
