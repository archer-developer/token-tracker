# Руководство по установке приложения

Это руководство описывает, как запустить и установить Трекер коммерческих токенов на вашем устройстве.

## Что такое PWA?

**PWA** (Progressive Web App — Прогрессивное веб-приложение) — это веб-приложение, которое работает как обычное приложение:

- ✅ Запускается одним кликом с экрана рабочего стола
- ✅ Работает **полностью офлайн** (после первой загрузки)
- ✅ Имеет значок на рабочем столе как обычное приложение
- ✅ Все данные хранятся **локально в браузере** (не на сервере)

## Быстрый старт

### Вариант 1: Запустить в браузере (самый быстрый)

1. Откройте в браузере: **https://token-tracker.example.com** (вставьте реальный URL)
2. Приложение загрузится и будет готово к использованию
3. Все данные будут сохраняться локально в браузере

### Вариант 2: Установить как приложение (рекомендуется)

Установка создает значок на рабочем столе и вы сможете запускать приложение как обычное ПО.

## Windows 10/11

### Способ 1: Chrome/Edge (Рекомендуется)

#### Через меню браузера

1. **Откройте приложение** в браузере Chrome или Edge
2. **Нажмите на иконку** в адресной строке (выглядит как 📥 или ⬇️)
3. Выберите **"Установить приложение"** или **"Install app"**
4. Подтвердите установку
5. Приложение появится на рабочем столе и в меню "Пуск"

#### Через меню браузера (если нет иконки в адресной строке)

1. Нажмите **три точки** ⋮ в углу браузера
2. Выберите **"Приложения"** → **"Установить это приложение"**
3. Подтвердите установку

### Способ 2: Firefox

Firefox пока не поддерживает установку PWA на Windows, но приложение будет работать в браузере:

1. Откройте приложение в Firefox
2. Закрепите вкладку: **Правый клик на вкладке** → **"Закрепить вкладку"**
3. Или создайте ярлык на рабочий стол:
   - **Правый клик на адресную строку** → **"Создать ярлык"**

### Способ 3: Ручное создание ярлыка

1. Откройте приложение в браузере
2. **Правый клик на страницу** → **"Создать ярлык"**
3. Выберите **"Рабочий стол"**
4. Теперь на рабочем столе появится ярлык для быстрого запуска

2. Нажмите на **три точки** ⋮
3. Выберите **"Apps"** → **"Install this app as window"**
4. Приложение появится в меню приложений и можно закрепить в панели

### Способ 2: Firefox

Firefox на Linux не поддерживает PWA в полной мере, но приложение работает:

1. Создайте файл `TokenTracker.desktop` на рабочем столе:

```ini
[Desktop Entry]
Name=Token Tracker
Exec=firefox https://token-tracker.example.com
Icon=web-browser
Type=Application
Categories=Finance;Office;
```

2. Сохраните файл и дайте ему права на исполнение:

```bash
chmod +x ~/Desktop/TokenTracker.desktop
```

## Первая загрузка и данные

### При первом запуске

1. Приложение загрузит все необходимые файлы (~5-10 MB)
2. Требуется **интернет соединение** для первой загрузки
3. После этого приложение работает **полностью офлайн**

### Где хранятся данные?

- **Windows/Mac/Linux**: Данные хранятся в папке браузера:
  - Chrome: `%LocalAppData%\Google\Chrome\User Data` (Windows)
  - Safari: `~/Library/Safari` (Mac)
- **Мобильные**: Данные в приложении браузера

Все данные остаются в браузере и **никогда не отправляются на сервер**.

## Обновления приложения

### Как работает обновление PWA

Приложение использует **Service Worker** для управления обновлениями. Вот как это работает:

#### Процесс обновления:

1. **При загрузке страницы** браузер запускает Service Worker
2. **Service Worker проверяет** наличие новой версии на сервере
3. **Если найдена новая версия:**
   - Скачивает все новые файлы в фоне (вы продолжаете работать)
   - Сохраняет их в локальном кэше браузера
4. **При следующем запуске:**
   - Приложение использует обновленную версию
   - Все ваши данные остаются на месте

#### Пример обновления:

```
Понедельник 10:00
└─ Вы открыли приложение (версия 1.0.0)
   Service Worker проверяет обновления на сервере

Понедельник 15:00
└─ Была выпущена версия 1.1.0
   Service Worker скачивает новые файлы в фоне
   Вы продолжаете работать с версией 1.0.0

Вторник 09:00
└─ Вы открываете приложение
   ✓ Используется версия 1.1.0
   ✓ Все ваши данные на месте
   ✓ Обновление было прозрачным (вы ничего не делали)
```

### Ручная проверка обновления

Если вы хотите получить обновление немедленно (не ждать следующего запуска):

**Все платформы:**

1. **Закройте приложение полностью** (закройте окно/вкладку)
2. **Откройте приложение заново**
3. Service Worker проверит обновления и загрузит новую версию (если есть)

**Альтернативный способ (Chrome):**

1. Откройте приложение
2. Нажмите на **три точки** ⋮ (если видно в окне приложения)
3. Нажмите **"Обновить"** (если доступно)

### Кэширование и хранилище

PWA приложение использует два типа хранилища:

#### 1. **Service Worker Cache** (кэш браузера)

- Хранит файлы приложения (HTML, CSS, JS, изображения)
- Позволяет работать **офлайн**
- Обновляется автоматически через Service Worker
- Размер: обычно 10-50 MB

#### 2. **IndexedDB** (локальная база данных)

- Хранит все ваши инвестиционные данные
- Не зависит от кэша приложения
- Остается **при обновлении** приложения
- Размер: зависит от количества ваших инструментов

**Важно:** При обновлении приложения обновляется только кэш Service Worker, ваши данные остаются нетронутыми!

### Очистка кэша (если есть проблемы)

Если приложение работает странно, попробуйте очистить кэш:

**Chrome/Edge:**

1. Откройте приложение
2. Нажмите на **три точки** ⋮
3. **Settings** → **Apps** → Token Tracker
4. **Storage** → **Clear browsing data**
5. Перезагрузите приложение

**Safari (Mac):**

1. **Safari** → **Preferences** → **Privacy**
2. **Manage Website Data**
3. Найдите приложение и нажмите **Remove**
4. Закройте Safari полностью
5. Откройте приложение заново

**Safari (iOS):**

1. **Настройки** → **Safari**
2. **Очистить историю и данные веб-сайтов**
3. Откройте приложение заново

**⚠️ Важно:** Очистка кэша **не удалит** ваши данные! Они хранятся отдельно в IndexedDB.

## Проблемы и решения

### Не вижу кнопку "Установить приложение"

**Причины:**

- Браузер не поддерживает PWA (используйте Chrome/Edge/Safari)
- Приложение загружается с HTTP вместо HTTPS (не поддерживается)

**Решение:**

- Убедитесь, что используете URL с `https://`
- Попробуйте другой браузер

### Приложение не загружается

**Решения:**

1. Проверьте интернет соединение
2. Очистите кэш браузера
3. Попробуйте другой браузер
4. Перезагрузите компьютер/телефон

### Данные исчезли после удаления приложения

К сожалению, при удалении приложения его локальное хранилище также удаляется.

**Как избежать потери данных:**

1. Перед удалением экспортируйте данные:
   - **Настройки** → **Резервная копия** → **Экспортировать**
2. Сохраните JSON файл
3. После переустановки импортируйте данные обратно:
   - **Настройки** → **Резервная копия** → **Импортировать**

### Приложение медленно загружается

**Причины:**

- Медленное интернет соединение
- Большое количество данных в приложении

**Решения:**

1. Подождите полной загрузки при первом запуске
2. После этого приложение будет работать быстро (офлайн)
3. Попробуйте другую сеть WiFi

## Безопасность и конфиденциальность

### Где хранятся мои данные?

✅ **Все данные хранятся локально** в браузере вашего устройства. Регулярно экспортируйте свои данные в файл и сохраняйте его в надежном месте  
✅ **Никаких серверов** — приложение не отправляет данные никуда  
✅ **Полная конфиденциальность** — только вы видите свои инвестиции

### HTTPS и шифрование

- Приложение работает только по HTTPS (защищенное соединение)
- При передаче файлов используется TLS/SSL шифрование
- Данные в браузере защищены механизмами браузера

### Что собирает приложение?

❌ Приложение **не собирает** никаких аналитик  
❌ **Не отслеживает** ваши действия  
❌ **Не использует** cookies для трекинга

## Для технических пользователей

### Как Service Worker обнаруживает обновления?

Приложение использует **хеширование файлов** для обнаружения обновлений:

#### Процесс:

1. **При сборке** каждому файлу приложения назначается уникальный хеш (например: `app.a1b2c3d4.js`)
2. **Service Worker загружает** список всех файлов с их хешами (`manifest.webmanifest`)
3. **При следующем запуске** Service Worker сравнивает хеши:
   - Если хеши совпадают — обновлений нет
   - Если хеши отличаются — скачивает новые файлы

#### Пример:

```
Версия 1.0.0:
├── app.a1b2c3d4.js (хеш файла)
├── styles.x9y8z7w6.css
└── manifest.webmanifest

Версия 1.1.0:
├── app.e5f6g7h8.js (новый хеш!)
├── styles.x9y8z7w6.css (хеш не изменился)
└── manifest.webmanifest

Service Worker видит:
✓ app.js изменился → скачивает новую версию
✓ styles.css не изменился → используется кэшированная версия
✓ manifest.webmanifest не изменился → используется кэшированная версия
```

### Стратегия кэширования

Приложение использует разные стратегии кэширования для разных ресурсов:

| Ресурс                       | Стратегия     | Описание                               |
| ---------------------------- | ------------- | -------------------------------------- |
| **Основные файлы (JS, CSS)** | Cache First   | Сначала использует кэш, потом интернет |
| **API NBRB**                 | Network First | Сначала пытается интернет, потом кэш   |
| **Изображения**              | Cache First   | Кэшируется на длительное время         |

Это означает:

- Приложение быстро загружается из кэша
- Курсы валют всегда актуальные (Network First)
- При отсутствии интернета используются кэшированные курсы

### Размер приложения

```
Service Worker Cache: ~10-15 MB
├── HTML/JS/CSS: ~2-3 MB
├── Шрифты: ~3-4 MB
├── Иконки: ~2-3 MB
└── Библиотеки: ~3-5 MB

IndexedDB (ваши данные): зависит от количества записей
```
