## Формула расчёта процентного дохода (п. 4.9)

```
СПД = Σ (СТВi × СД/100 × 1/КДГ),  i = 1..ДВ
```

Где:

- **СПД** — сумма процентного дохода за период
- **СД** = 11 (ставка дохода, % годовых)
- **КДГ** — количество дней в календарном году, в котором находится соответствующий период: **365** (366 в високосном году)
- **СТВi** — сумма вложенных в токены средств на день i
- **ДВ** — количество дней владения токеном

При постоянном остатке токенов в течение периода формула сворачивается до:

```
СПД = principal × (СД / 100) × daysInPeriod / KDG
```

### Правила округления

Округление итоговой суммы за период — математическое, до 2 знаков после запятой:

- если **3-я** цифра после запятой **≥ 5** → 2-я цифра увеличивается на 1
- если **3-я** цифра после запятой **< 5** → без изменений

Это стандартное «round half up».

### Что НЕ учитывается текущим (неправильным) алгоритмом

Текущий код использует:

```ts
couponPerPeriod = (principal * annualRate) / periodsPerYear // ≈ principal * 0.11 / 12
```

Это **неверно**, потому что:

1. Месяцы имеют разное количество дней (28–31)
2. Год может быть високосным (366 дней) или обычным (365)
3. Первый период (30.12.2024 – 31.12.2024) — всего 2 дня
